===========reveter o ultimo commit do github(servidor)==================

git push origin HEAD^:branch-name --force =>
Este comando força a branch no repositório remoto a apontar para o commit anterior ao HEAD (o commit atual). Isso efetivamente remove o último commit do histórico remoto. 

=======desfazer o ultimo commit ======

Desfazer o último commit localmente (ainda não enviado ao servidor): git reset --soft HEAD~1 
Desfazer o último commit localmente, descartando mudanças: git reset --hard HEAD~1
Desfazer o último commit já enviado ao servidor: git revert HEAD
Alterar o último commit (após corrigir arquivos):
git add .
git commit --amend

=====git stash=========
git stash = O Git tem uma área chamada stash (algo como 'esconderijo', em português), onde você pode armazenar temporariamente um instantâneo de suas alterações sem fazer o commit delas para o repositório. 

1. Aplicar as alterações de volta ao branch:
git stash apply => Isso aplicará as alterações do stash ao seu branch, mas não removerá o stash. 
git stash drop =>  Se você quiser remover o stash após aplicar as alterações .
git stash pop => Se você quiser aplicar e remover o stash em um único comando.

2. Excluir as alterações do stash sem aplicá-las:

git stash drop => Isso removerá as alterações do stash sem aplicá-las ao seu branch.
git stash drop stash@{n} => Se você tiver vários stashes e quiser remover um stash específico 
Onde n é o índice do stash que você deseja remover.
